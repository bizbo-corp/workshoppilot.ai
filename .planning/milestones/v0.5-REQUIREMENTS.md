# Requirements: WorkshopPilot.ai

**Defined:** 2026-02-07
**Core Value:** Anyone with a vague idea can produce validated, AI-ready product specs without design thinking knowledge.

## v1 Requirements

Requirements for MVP 0.5 — Application Shell. Each maps to roadmap phases.

### Authentication

- [x] **AUTH-01**: User can sign up and sign in via Clerk (email + password)
- [x] **AUTH-02**: System supports two roles: facilitator and participant
- [x] **AUTH-03**: Facilitator can access admin/dashboard routes; participant cannot
- [x] **AUTH-04**: Protected routes redirect unauthenticated users to sign-in
- [x] **AUTH-05**: User session persists across browser refresh

### Application Scaffolding

- [x] **SCAF-01**: App shell with header (logo, user menu), sidebar (step list), and main content area
- [x] **SCAF-02**: Desktop-first layout with consistent navigation across all pages
- [x] **SCAF-03**: Route structure follows `/workshop/[sessionId]/step/[stepId]` pattern
- [x] **SCAF-04**: Landing page with clear "Start Workshop" entry point

### Workshop Steps

- [x] **STEP-01**: All 10 design thinking steps exist as routable pages with titles
- [x] **STEP-02**: Each step displays its name, number, and placeholder instructions
- [x] **STEP-03**: Step content area is ready to receive AI chat and form components (hollow container)

### Navigation & Progress

- [x] **NAV-01**: Linear stepper/progress bar shows all 10 steps with current position
- [x] **NAV-02**: Steps display states: complete (checkmark), current (highlighted), upcoming (disabled)
- [x] **NAV-03**: "Next" button advances to the next step
- [x] **NAV-04**: "Back" button returns to the previous step
- [x] **NAV-05**: Sequential enforcement — user cannot skip ahead to uncompleted steps

### Database

- [x] **DATA-01**: Neon Postgres database provisioned and connected
- [x] **DATA-02**: Drizzle ORM configured with schema definition and migration workflow
- [x] **DATA-03**: Database connection works in Vercel serverless environment (edge-compatible driver)

### AI Chat Integration

- [x] **CHAT-01**: Gemini API connected via Vercel AI SDK with streaming responses
- [x] **CHAT-02**: Basic chat interface component (message input, send button, message history)
- [x] **CHAT-03**: Chat UI displays at each step (same generic AI, not yet step-specific)
- [x] **CHAT-04**: AI responses stream in real-time (typing indicator, progressive display)

### Deployment

- [x] **DEPLOY-01**: Application deploys to Vercel from main branch
- [x] **DEPLOY-02**: Environment variables configured for Clerk, Neon, and Gemini API keys
- [x] **DEPLOY-03**: Application accessible via workshoppilot.ai domain
