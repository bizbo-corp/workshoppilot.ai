  <action>
Create `src/components/workshop/mobile-gate.tsx` as a `'use client'` component.

**Detection logic (in useEffect only — SSR-safe):**
- Initialize `const [show, setShow] = useState(false)` — server always renders null
- In `useEffect`, first check sessionStorage for `wp_mobile_gate_dismissed` key (try/catch wrapped). If `'1'`, return early — already dismissed
- Then check `window.matchMedia('(pointer: coarse) and (max-width: 1023px)')`. If `.matches` is true, call `setShow(true)`
- No resize/matchMedia event listener — detection is intentionally one-shot at mount per CONTEXT.md locked decision ("no special mobile layout after dismissal")

**Dismiss handler:**
- `dismiss()` function: sets `sessionStorage.setItem('wp_mobile_gate_dismissed', '1')` (try/catch wrapped), then `setShow(false)`
- Wired to the "Continue anyway" text link

**Props:**
- Accept `workshopName?: string` prop (used in mailto subject line)

**Layout and styling (locked decisions from CONTEXT.md):**
- Full-screen edge-to-edge overlay: `fixed inset-0 z-[200]` (above session-ended at z-[100])
- Solid background using brand palette: `bg-neutral-olive-950` (dark) — NOT semi-transparent, NOT backdrop-blur
- Flex column, items-center, justify-center, with `px-6` padding for content centering
- Max-width container for content: `max-w-sm` to keep text readable

**Illustration (Claude's discretion):**
- Create an inline SVG of a simplified laptop/monitor outline in olive-300/olive-400 tones
- Approximately `w-24 h-24` — large enough to be the visual anchor of the overlay
- Simple, geometric, flat style — matches the brand's clean aesthetic
- Alternative: use Lucide `Monitor` icon at large size if inline SVG feels over-engineered. Either approach is fine.

**Copy:**
- Headline: "WorkshopPilot works best on desktop" — use `text-xl font-semibold text-olive-50` or similar light-on-dark
- Body: One sentence explaining why, e.g. "The canvas and AI tools need a larger screen for the best experience." — use `text-sm text-olive-300`
- Tone: confident, helpful, NOT apologetic per CONTEXT.md

**CTAs (locked hierarchy from CONTEXT.md):**
1. Primary: "Email this link to myself" button
   - Styled as a prominent button (light on dark background — `bg-olive-100 text-olive-900 hover:bg-olive-200` or use the Button component with appropriate variant)
   - `href` = `mailto:?subject=${encodeURIComponent(subject)}&body=${encodeURIComponent(body)}`
   - Subject: `Continue your workshop on desktop — WorkshopPilot` (include workshopName prop if available: `Continue "${workshopName}" on desktop — WorkshopPilot`)
   - Body: `Open this link on your desktop to continue your workshop:\n\n${window.location.href}`
   - Use an `<a>` tag with `href` (not onClick), so the device's mail app opens natively

2. Secondary: "Copy link" — small text/icon option below the email button
   - Use `navigator.clipboard.writeText(window.location.href)` in an async onClick handler
   - Track `copied` state: show "Copied!" for 2 seconds after successful copy, then revert
   - Try/catch wrap — if clipboard API fails, silently degrade (no toast — overlay is already showing)
   - Use Lucide `Copy` and `Check` icons for visual feedback: `Copy` icon normally, `Check` icon when copied

3. Tertiary: "Continue anyway" — secondary text link at the bottom
   - Plain text link style: `text-sm text-olive-400 underline underline-offset-4 hover:text-olive-300`
   - Calls `dismiss()` on click

**Animation (Claude's discretion):**
- Use framer-motion `AnimatePresence` + `motion.div` for entry/exit
- Entry: `initial={{ opacity: 0 }}` → `animate={{ opacity: 1 }}` with `duration: 0.3`
- Exit: `exit={{ opacity: 0 }}` with `duration: 0.2`
- Keep it simple — this is a gate, not a showcase

**Important — do NOT:**
- Use the existing `useIsMobile` hook from `src/hooks/use-mobile.ts` (wrong breakpoint, no pointer check)
- Use `localStorage` instead of `sessionStorage` (locked decision)
- Use `navigator.userAgent` for detection (unreliable, outdated)
- Add a resize listener (locked: no special mobile layout after dismissal)
- Render anything during SSR (useState(false) + useEffect pattern)

To:
```tsx

  <>
    <MobileGate workshopName={session.workshop.title || 'New Workshop'} />

      {/* ... existing content unchanged ... */}
    </SidebarProvider>
  </>

```

**Why outside SidebarProvider:** The `SidebarProvider` may create a stacking context via transform/will-change CSS. Placing `MobileGate` as a sibling ensures `fixed inset-0 z-[200]` correctly overlays everything. This matches the research recommendation from 62-RESEARCH.md Pitfall 6.

**Scope check (MOBI-03):** This layout wraps ONLY `/workshop/[sessionId]/*` routes (step pages, results, outputs). Landing page (`/`), dashboard (`/dashboard`), and pricing (`/pricing`) use different layouts and are NOT affected. No additional route filtering is needed — the layout scope IS the gate scope.

**Do NOT modify any other part of the layout.** The only changes are the import line and wrapping the return in a Fragment with MobileGate.
