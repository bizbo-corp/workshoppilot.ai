  <action>
Create `src/lib/canvas/voting-types.ts` following the pattern established by `crazy-8s-types.ts`. The file defines three types and one constant:

1. **`DotVote` type** — a single vote cast by a participant:
   - `id: string` — `crypto.randomUUID()` assigned at cast time
   - `slotId: string` — which Crazy 8s slot received the vote (e.g., 'slot-3')
   - `voterId: string` — Liveblocks userId ('user_xxx' for owners, 'guest-xxx' for participants). Enables per-voter retraction (VOTE-06) and completion tracking (Phase 61)
   - `voteIndex: number` — 0-based index within this voter's budget, for retraction by position

2. **`VotingResult` type** — a ranked result entry populated when voting closes:
   - `slotId: string`
   - `totalVotes: number`
   - `rank: number` — 1 = most votes; ties share rank (VOTE-07)

3. **`VotingSession` type** — session-scoped voting configuration and status:
   - `status: 'idle' | 'open' | 'closed'` — 'idle' = not yet opened, 'open' = voting active, 'closed' = results visible (VOTE-05)
   - `voteBudget: number` — votes per participant; default 2, range 1-8 (VOTE-01 per STATE.md decision)
   - `results: VotingResult[]` — populated by setVotingResults(); empty while idle/open (VOTE-07)

4. **`DEFAULT_VOTING_SESSION` constant** — initial state:
   ```typescript
   export const DEFAULT_VOTING_SESSION: VotingSession = {
     status: 'idle',
     voteBudget: 2, // NNGroup 25%-of-options rule: 8 slots x 25% = 2 (STATE.md decision)
     results: [],
   };
   ```

Use `export type` for all three types (not `interface`) to match the existing `ConceptCardData`, `PersonaTemplateData`, `HmwCardData` pattern in this project. Include JSDoc comments on each type explaining its purpose and field semantics.

Important: Do NOT add any `isVisible` or `isAnonymous` field to `DotVote`. VOTE-06 anonymity is a UI rendering concern handled in Phase 60/61, not a data model concern.

**Step 6: Add action implementations** inside the `temporal((set) => ({...}))` block, after `toggleBrainRewritingIncluded` and before `markClean`:

```typescript
castVote: (vote) =>
  set((state) => ({
    dotVotes: [
      ...state.dotVotes,
      { ...vote, id: crypto.randomUUID() },
    ],
    isDirty: true,
  })),

retractVote: (voteId) =>
  set((state) => ({
    dotVotes: state.dotVotes.filter((v) => v.id !== voteId),
    isDirty: true,
  })),

openVoting: (voteBudget) =>
  set((state) => ({
    votingSession: {
      ...state.votingSession,
      status: 'open' as const,
      voteBudget: voteBudget ?? state.votingSession.voteBudget,
    },
    isDirty: true,
  })),

closeVoting: () =>
  set((state) => ({
    votingSession: { ...state.votingSession, status: 'closed' as const },
    isDirty: true,
  })),

setVotingResults: (results) =>
  set((state) => ({
    votingSession: { ...state.votingSession, results },
    isDirty: true,
  })),

resetVoting: () =>
  set(() => ({
    dotVotes: [],
    votingSession: DEFAULT_VOTING_SESSION,
    isDirty: true,
  })),
```

**Step 7: Add to `temporal()` partialize config** (after `brainRewritingMatrices: state.brainRewritingMatrices,`):
```typescript

votingSession: state.votingSession,
```

Important: All voting actions that mutate user data MUST set `isDirty: true`. The `openVoting` action uses `voteBudget ?? state.votingSession.voteBudget` so callers can omit it to keep the current budget. Use `as const` on string literal assignments for status to satisfy TypeScript's narrowing.
