       - `createClient` import from `@liveblocks/client` with `authEndpoint: "/api/liveblocks-auth"`
       - Export `liveblocksClient` (the client instance)
       - Export `getRoomId(workshopId: string): string` that returns `workshop-${workshopId}`
       - `declare global { interface Liveblocks { ... } }` with:
         - **Presence**: `{ cursor: { x: number; y: number } | null; color: string; displayName: string; }`
           - cursor is null when mouse leaves canvas (per CONTEXT.md: cursor position only)
           - color is the participant's assigned hex color
           - displayName from Clerk or guest name entry
         - **Storage**: `{ elements: LiveMap<string, LiveObject<CanvasElementStorable>> }`
           - Define `CanvasElementStorable` type (JSON-safe subset of canvas types) with:
             `{ id: string; type: string; position: { x: number; y: number }; width: number; height: number; data: Record<string, unknown>; authorId: string; }`
           - Use `LiveMap<string, LiveObject<...>>` for ID-keyed canvas elements
           - LiveObject wraps each element for granular undo tracking (per CONTEXT.md: per-user undo/redo)
         - **UserMeta**: `{ id: string; info: { name: string; color: string; role: 'owner' | 'participant'; } }`
           - `role` replaces `isOwner` boolean for cleaner role checks in later phases

    3. Do NOT install `@liveblocks/zustand` — that is Phase 55's concern.
    4. Do NOT create any React components or providers — that is Phase 55's concern.
    5. Do NOT use `createRoomContext()` — it was removed in Liveblocks v2.0. Use global interface augmentation only.
