    **Step 2 — Create `src/stores/multiplayer-canvas-store.ts`:**
    This file exports `createMultiplayerCanvasStore` — a factory that wraps the same state shape and actions as `createCanvasStore` with `liveblocks()` as the outermost middleware and `temporal()` as inner middleware.

    Structure:
    ```typescript
    import { createStore } from 'zustand/vanilla';
    import { liveblocks, type WithLiveblocks } from '@liveblocks/zustand';
    import { temporal } from 'zundo';
    import { liveblocksClient } from '@/lib/liveblocks/config';
    import type { CanvasStore, CanvasState } from './canvas-store';
    // ... import all types used by CanvasStore
    ```

    **Middleware composition:** `liveblocks(temporal((set, get) => ({ ...state, ...actions }), temporalOptions), liveblocksOptions)`

    - `liveblocks()` is outermost — it intercepts `set()` to detect mapped state changes for sync
    - `temporal()` is inner — provides undo/redo on local state
    - If TypeScript rejects the composition (type constraint issue), fall back to `liveblocks()` only (remove `temporal`) for multiplayer. Add a comment: "temporal removed — liveblocks+temporal composition failed, undo/redo disabled for multiplayer per STATE.md fallback decision"

    **storageMapping (durable fields only):**
    ```typescript
    storageMapping: {
      stickyNotes: true,
      drawingNodes: true,
      gridColumns: true,
      crazy8sSlots: true,
      mindMapNodes: true,
      mindMapEdges: true,
      conceptCards: true,
      personaTemplates: true,
      hmwCards: true,
      selectedSlotIds: true,
      brainRewritingMatrices: true,
    }
    ```

    **Ephemeral fields that MUST NOT be in storageMapping:** `isDirty`, `selectedStickyNoteIds`, `highlightedCell`, `pendingFitView`, `pendingHmwChipSelection`. These are per-client and would thrash between participants.

    **temporal partialize:** Same list as storageMapping (only durable state enters undo history).

    **Client:** Pass `liveblocksClient` from `@/lib/liveblocks/config`.

    **Actions:** Copy the full action implementations from `createCanvasStore` (lines ~159-600 of canvas-store.ts). In multiplayer mode, `markDirty()` and `markClean()` become no-ops (Liveblocks handles persistence, not the auto-save hook). Set `isDirty` to always be `false` in the multiplayer store.

    **Init state parameter:** Same shape as `createCanvasStore`'s initState param. The `DEFAULT_STATE` initialization is identical.

    **Export type:** `export type MultiplayerCanvasStoreApi = ReturnType<typeof createMultiplayerCanvasStore>;`
