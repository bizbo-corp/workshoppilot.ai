**A. `journey-flow-content.tsx`:**

2. State in `JourneyFlowInner`: `const [isGenerating, setIsGenerating] = useState(false);` and `const [showRegenerateConfirm, setShowRegenerateConfirm] = useState(false);`. Select `flowArchetype` from the store (`useJourneyFlowStore((s) => s.flowArchetype)`).
3. `handleGenerate = useCallback(async (scope: TestScope, selectedConceptName?: string, force = false) => {...})`:
   - Clear the autosave timer FIRST (`if (saveTimerRef.current) { clearTimeout(saveTimerRef.current); saveTimerRef.current = null; }`) — Pitfall 2: a pending debounce must never fire mid-replace.
   - `setIsGenerating(true)`; `fetch('/api/build-pack/generate-journey-flow', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ workshopId, scope, selectedConceptId: selectedConceptName, force }) })`.
   - On `!res.ok`: parse error body, `toast.error(message || 'Generation failed — please try again')` (sonner import), keep current state.
   - On success: `const { state } = await res.json();` then `storeApi.setState({ nodes: state.nodes, edges: state.edges, isApproved: false, isDirty: false, flowArchetype: state.flowArchetype, strategicIntent: state.strategicIntent, testScope: state.testScope, selectedConceptId: state.selectedConceptId, lastGeneratedAt: state.lastGeneratedAt, isTwoSided: state.isTwoSided })` — the route already persisted, so `isDirty: false` prevents a redundant autosave. `toast.success('Baseline flow generated')`.
   - `finally { setIsGenerating(false); }`
4. `handleRegenerate = () => setShowRegenerateConfirm(true)` — ALWAYS confirm (simplest safe UX; regeneration clobbers any user edits). `executeRegenerate` reads stored scope: `const s = storeApi.getState(); handleGenerate(s.testScope ?? 'journey', s.selectedConceptId, true);` then closes the dialog. (If `testScope` is missing — e.g. a flow started from scratch — default to `'journey'`.)
5. Render branches in `JourneyFlowInner`:
   - `if (!hasNodes && isReadOnly)`: keep the current read-only message block (workflow icon + "No journey flow has been created yet." + back link).
   - `if (!hasNodes && isGenerating)`: centered generating state — spinner icon `animate-spin` + muted "Generating your baseline flow…" text.
   - `if (!hasNodes)`: render `<ScopeChooser concepts={concepts} isGenerating={isGenerating} onGenerate={(scope, name) => handleGenerate(scope, name)} onStartFromScratch={handleAddFirstScreen} />` wrapped with the existing back-link (keep "Back to Build Pack" reachable below the chooser, or include it in the chooser layout).
   - else: `<JourneyFlowCanvas workshopId={workshopId} sessionId={sessionId} isReadOnly={isReadOnly} onRegenerate={isReadOnly ? undefined : handleRegenerate} isGenerating={isGenerating} />`.
6. Regenerate `AlertDialog` (shadcn, same composition as `journey-map-content.tsx`'s reset dialog): title "Regenerate baseline flow?", description "This will replace all current screens and connections with a fresh AI-generated baseline. Any edits you've made will be lost.", Cancel + destructive-styled action "Regenerate" → `executeRegenerate`. Render it alongside the canvas branch (controlled by `showRegenerateConfirm`).
7. Delete nothing else — autosave effect, store provider wrapper, and `handleAddFirstScreen` stay as-is.
