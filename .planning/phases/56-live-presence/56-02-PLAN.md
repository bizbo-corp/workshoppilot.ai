**1. `useIdleStatus` custom hook:**

Tracks idle state for each connected participant. A participant is "idle" if they haven't moved their cursor for 2+ minutes.

```typescript
const IDLE_THRESHOLD_MS = 2 * 60 * 1000; // 2 minutes

function useIdleStatus(): Set<number> {
  const lastSeenRef = useRef<Map<number, number>>(new Map());
  const [idleIds, setIdleIds] = useState<Set<number>>(new Set());

  useOthersListener(({ type, user }) => {
    if (type === 'enter') {
      // Treat new/reconnected users as freshly active
      lastSeenRef.current.set(user.connectionId, Date.now());
    } else if (type === 'leave') {
      lastSeenRef.current.delete(user.connectionId);
    } else if (type === 'update') {
      // Any presence update (cursor move, etc.) counts as activity
      if (user.presence?.cursor !== null && user.presence?.cursor !== undefined) {
        lastSeenRef.current.set(user.connectionId, Date.now());
      }
    }
  });

  // Re-evaluate idle status every 30 seconds
  useEffect(() => {
    const interval = setInterval(() => {
      const now = Date.now();
      const newIdleIds = new Set<number>();
      lastSeenRef.current.forEach((lastSeen, connectionId) => {
        if (now - lastSeen > IDLE_THRESHOLD_MS) {
          newIdleIds.add(connectionId);
        }
      });
      setIdleIds(newIdleIds);
    }, 30_000);
    return () => clearInterval(interval);
  }, []);

  return idleIds;
}
```

This is a renderless component that fires toasts when participants join or leave.

```typescript
function JoinLeaveListener() {
  useOthersListener(({ type, user }) => {
    if (type === 'enter') {
      toast(`${user.info?.name ?? 'Someone'} joined`, {
        duration: 3000,
      });
    } else if (type === 'leave') {
      toast(`${user.info?.name ?? 'Someone'} left`, {
        duration: 3000,
      });
    }
  });
  return null;
}
```

**Design notes (per CONTEXT.md):**
- Uses plain `toast()` (not `toast.success` or `toast.error`) for neutral join/leave notifications.
- `duration: 3000` = 3 second auto-dismiss per context decision.
- The `Toaster` is already rendered at `position="bottom-right"` in `app/layout.tsx` — no position override needed.
- Each person gets their own toast, no batching.
- Same toast style for all roles — no special facilitator emphasis per context decision.
