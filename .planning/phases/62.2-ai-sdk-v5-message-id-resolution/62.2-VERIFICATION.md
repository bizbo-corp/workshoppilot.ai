### Human Verification Required

#### 1. 5x Rapid-Mount Greeting Idempotency

**Test:** Open a workshop step in a browser with React Strict Mode enabled (development build). Trigger the step start 5 times in rapid succession (or open 5 tabs to the same step simultaneously).
**Expected:** Exactly 1 `chat_messages` row with `role='assistant'` exists in the DB for that session+step+participant scope. No `greeting:…` prefixed `messageId` remains after fill completes.
**Why human:** The concurrent-claim race (two concurrent `claimGreetingPlaceholder` inserts resolving via unique index) cannot be exercised deterministically in a script. Requires real browser timing with React's double-invoke in Strict Mode.

#### 2. Race-Loser Replay Client Receipt

**Test:** Open two browser tabs to the same workshop step simultaneously so they race on the greeting claim. Confirm the losing tab receives the greeting content via the replay stream (not a second AI generation).
**Expected:** Losing tab shows the same greeting text as the winner. DB has exactly 1 `chat_request_logs` row with `is_replay=false` and 1 row with `is_replay=true`, both with `response_message_id` populated. No extra `chat_messages` assistant rows.
**Why human:** Requires coordinated multi-tab timing that cannot be scripted against a running dev server without custom concurrency harness.
