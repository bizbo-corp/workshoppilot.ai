## How to Use

1. Each major feature gets its own spec folder or file
2. Link specs to backlog items and sprint tasks
3. Reference specs in GitHub issues
4. Keep specs updated as requirements evolve

## Spec Template

```markdown
# [Number]: [Feature Name]

## Overview
**Status**: Draft | In Review | Approved | In Development | Completed
**Priority**: P0 | P1 | P2 | P3
**Target Release**: MVP 0.5 | MVP 1.0 | v2.0
**Owner**: @username
**Dev Repo**: Issue #[X]

## Problem Statement
[What problem are we solving?]

## Goals & Success Metrics
- Goal 1: [Metric]

## User Stories
- As a [role], I want [action] so that [benefit]

## Requirements
### Must Have
- [ ] Requirement 1

### Nice to Have
- [ ] Enhancement 1

## Technical Considerations
[Architecture, dependencies, constraints]

## Design Notes
[UI/UX considerations, mockups]

## Testing Plan
[How we'll validate this works]

## Dev Repo Reference
**Repository**: [GitHub URL]
**Branch**: feature/[name]
**Epic/Issue**: #[X]

## Related Documents
- [[link to other specs]]

## Open Questions
- [ ] Question 1

## Updates Log
| Date | Update | By |
|------|--------|-----|
```
